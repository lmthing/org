# Explore the Monorepo

Give the developer a guided tour of the monorepo, explaining the TLD-based organization, shared libraries, and how packages relate to each other.

## Key Areas to Cover

1. **TLD organization** — each `lmthing.*` domain has its own directory
2. **Shared libraries** in `org/libs/` — the foundation everything depends on
3. **Cloud backend** in `cloud/` — the sole backend for all products
4. **Package relationships** — what depends on what, and how imports resolve
5. **Configuration files** — `pnpm-workspace.yaml`, `services.yaml`, Vite configs

## Tour Order

Start with the high-level layout, then drill into `org/libs/` (most important for understanding the system), then `cloud/`, then show a couple of product apps as examples.
