# Make Your First Change

Walk the developer through a small, meaningful code change to build confidence with the codebase. The change should touch at least one shared lib and one product app.

## Suggested First Changes

- Add a new UI component to `@lmthing/ui` and use it in Studio
- Add a new knowledge option to an existing space
- Create a simple utility function in a shared lib
- Add a new route to a product app

## Steps

1. Pick a change based on the developer's interests and skill level
2. Identify which packages are involved
3. Make the change with guidance on conventions
4. Verify it works in the dev environment (HMR should pick it up)
5. Explain the contribution workflow (branch, commit, PR)
