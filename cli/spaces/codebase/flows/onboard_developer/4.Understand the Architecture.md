# Understand the Architecture

Walk through the complete data flow for a typical user interaction — sending a message to an agent in Studio — to understand how all the pieces connect.

## The Flow to Trace

```
User sends message → Studio reads agent config from VFS → POST to generate-ai →
Edge function authenticates → Stripe LLM proxy → Provider → Token stream back →
SSE to browser → Real-time rendering
```

## Key Concepts to Explain

1. **No separate backend** — all server logic is edge functions in `cloud/`
2. **VFS as data layer** — workspace files live in-memory, synced to GitHub
3. **Stripe as billing proxy** — all LLM requests go through Stripe for metering
4. **Auth is centralized** — all apps authenticate through `com/`
5. **Spaces are self-contained** — agents, flows, and knowledge work together
