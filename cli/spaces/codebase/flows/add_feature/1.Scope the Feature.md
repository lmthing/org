# Scope the Feature

Before writing code, understand what the feature requires and plan the implementation across the monorepo.

## Analysis Steps

1. **Categorize the feature** — is it frontend-only, backend-only, or full-stack?
2. **Identify affected packages** — which shared libs, apps, or cloud functions need changes?
3. **Check dependencies** — will changes to a shared lib affect other consumers?
4. **Identify patterns** — which existing patterns should this feature follow?
5. **Estimate complexity** — is this a single-file change or a cross-cutting concern?

## Decision Tree

- **New UI feature in one app** → modify that app only, use `@lmthing/ui` components
- **New shared component** → add to `@lmthing/ui`, export, use in target app
- **New API endpoint** → add edge function in `cloud/`, call from frontend
- **New agent capability** → modify `org/libs/core/` (framework) or add to a space
- **New space** → create under `org/libs/thing/spaces/` following space architecture
- **New product app** → create TLD directory, add to `services.yaml`, set up proxy
