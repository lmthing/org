# Ship It

Create a pull request and prepare the change for review.

## Steps

1. Stage and commit changes with a clear commit message
2. Push the branch to the remote
3. Create a PR with a descriptive title and summary
4. Review the diff one more time for any issues

## Commit Message Convention

```
type(scope): short description

- Detail about what changed and why
- Another detail if needed
```

Types: `feat`, `fix`, `refactor`, `docs`, `chore`, `style`, `test`
Scopes: package name (`core`, `state`, `ui`, `cloud`, `studio`, etc.)

## PR Description Template

- **Summary**: What this PR does and why
- **Changes**: List of key changes
- **Testing**: How it was tested
- **Screenshots**: If UI changes are involved
