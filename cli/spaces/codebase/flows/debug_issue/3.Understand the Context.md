# Understand the Context

Before fixing the bug, understand the code around it — why was it written this way? What else depends on it?

## Steps

1. **Read the full file** — understand the function/component's purpose
2. **Check callers** — who calls this code? What assumptions do they make?
3. **Check dependents** — if this is in a shared lib, which apps consume it?
4. **Read git history** — when was this code last changed? What was the intent?
5. **Identify constraints** — are there architectural reasons for the current approach?

## Context Checklist

- Is this a shared lib that multiple apps depend on? (Higher risk change)
- Is this an edge function with RLS implications? (Security consideration)
- Is this a state management issue? (VFS event propagation)
- Is this an auth flow issue? (SSO redirect chain)
- Is this a Fly.io container issue? (Infrastructure concern)
